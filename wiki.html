!!mobiletracker.py
This script downloads data from the mobile tracker feature of the metro web site. The data that we get is the url for (stop, route) pairs. The url allows us to get real-time arrival estimate for a (stopid, route). We also get a headsign for (stopid, route)s. The headsign says which way the bus is going (and also if it is a an alternate, via, route).

This data gets saved to {{{mobiletracker-weekdays.json}}} and {{{mobiletracker-weekendsandholidays.json}}}.

The syntax:
{{{
{
    stop_id: [{
        "routeno": route_number,
        "direction": the headsign,
        "url": url
    }]
}
}}}

!!get-stops.py
This script downloads data from the bus stop tables at http://www.cityofmadison.com/metro/BusStopDepartures/ The data that is obtained is for (stopid)s. We get the cardinal direction (e.g. northbound), the intersection where the stop is located and the (lat, lng) location (although it is missing for many stops).

This data is saved to file {{{stops.json}}}.

The syntax is
{{{
{
    stop_id: {
        "addr1": "TENNYSON LN", 
        "addr2": "ELIOT LN", 
        "dir": "EB", 
        "houseno": "1899", 
        "lat": "43.075831", 
        "lon": "-89.399024", 
    }
}
}}}
